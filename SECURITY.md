# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please:

1. **Do NOT** open a public issue
2. Email the details to the maintainer (or open a private security advisory on GitHub)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if you have one)

## What to Report

- API key exposure
- Code injection vulnerabilities
- Authentication bypasses
- Any security-related bugs

## What NOT to Report

- Bugs that don't have security implications (use regular issues)
- Feature requests

## Response Time

I'll try to respond within 48 hours and provide a fix as soon as possible.

## Supported Versions

Only the latest version is supported with security updates.

## Safe Practices

When using this project:
- Never commit your `.env` file
- Keep your API keys private
- Don't share your Groq API key
- Use environment variables for sensitive data
